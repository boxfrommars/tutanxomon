
Установка
=========

1. копируем на сервер папку tutanxomon
2. прописываем в vhost папку application (мой вариант файла: deploy/tutanxomon)
3. ставим права 777 на папку application/protected/runtime
4. создаём базу данных (utf8)
5. открываем файл application/config/main.php
	а. прописываем в отдел db правильные параметры соединения
	б. заодно меняем параметры входа админа rpcAdmin
6. выполняем в нашей бд sql из файла deploy/install.sql


Использование
=============

заходим на сайт, радуемся, кликаем подарить единорога, заполняем поздравление, нажимаем отправить
модальное окно закрыается, появляется бабл с нашим добавленным пожеланием 

наводим на любого подсвеченного единорога -- появляется бабл с пожеланием

кликаем на значок копирайта снизу справа, смотрим, закрываем


Администрирование
=================

заходим на /admin
клацаем ~ (тильда), появляется консоль, вводим логин и пароль, которые поменяли в rpcAdmin 
печатаем help
печатаем help нужная команда (например help delete) смотрим, что мы можем натворить. творим
печатаем exit
клацаем тильда и ложимся спать


Заметки о реализации
====================

1. принцип неопределённости гейзенберга

так как мониторы у всех разные, но при этом не хочется делать сайту фиксированную ширину, то мы не можем сохранять 
точную позицию пожелания-единорога (так как у некоторых он просто не влезет в экран, если будет достаточно сдвинут, например, вправо)
поэтому я поделил страницу на блоки (количество блоков определяется в application/js/script.js) и единороги привязываются
только к блоку, внутри же блока распределены случайно.
это даёт во-первых, примерное позиционирование единорога на том же месте (для удобного поиска) во-вторых, то, что единороги будут 
равномерно и примерно одинаково размазаны для любого монитора

2. терминал

просто пришла такая идея в голову, в любом случае, нужно иметь возможность удалять "неугодные пожелания", а писать занудную
админку не хотелось. вот и придумал такое извращение:

	открыть/закрыть консоль: ~ (тильда, там же где и ё)
	команды отправляются по enter, как это ни странно :)
	
	а. удалить пост по id: delete id (например delete 7)
	б. список последник 10 пожеланий: ls
	в. список последних x пожеланий: ls x (например ls 17)
	г. список последних x пожеланий, начиная с y-го от конца: ls x y (например ls 13 7)
	д. всего пожеланий: count
	е. разлогиниться: exit
	ж. список удалённых команд: help
	з. справка о команде command: help command (например help delete)
